# Gemini Chrome Agent

The Gemini Chrome Agent is a powerful, low-latency bridge that connects the Google Gemini web interface directly to your local Ubuntu environment. It enables Gemini to act as an autonomous agent capable of executing shell commands, managing the local filesystem, and performing Git operations directly from the chat interface.

---

## System Overview

The project consists of three main layers:

| Layer | Component | Purpose |
|-------|-----------|---------|
| **Frontend** | Chrome Extension | Scans Gemini responses for specific JSON payloads and injects system execution results back into the chat. |
| **Middleware** | Native Messaging | A secure communication channel between Brave/Chrome and the local OS. |
| **Backend** | Python Host | A persistent Python script that handles command execution, file I/O, and logging. |

### File Structure

| File | Description |
|------|-------------|
| `manifest.json` | The extension configuration, defining permissions (`nativeMessaging`, `scripting`) and the content script entry point. |
| `background.js` | The service worker that maintains a persistent connection to the Python host and routes messages between the browser and the system. |
| `content.js` | The "brain" inside the Gemini tab. It handles payload detection, result injection, and anti-looping logic. |
| `host.py` | The Python execution engine. It captures stdout, stderr, and exit codes, returning them as JSON objects. |
| `main.py` | A standalone CLI utility for testing agent actions without the browser extension. |
| `setup.sh` | An automated configuration script that handles permissions, Extension ID registration, and Native Messaging manifest symlinking. |
| `com.local.gemini_agent.json` | The registry file that tells the browser how to launch the Python host. |

## Installation & Setup

### 1. Prepare the Extension

1. Open Brave and navigate to `brave://extensions/`.
2. Enable **Developer mode** (toggle in the top-right corner).
3. Click **Load unpacked** and select the `~/Development/Gemini-Chrome-Agent` directory.
4. Copy the **Extension ID** generated by Brave.

### 2. Run the Setup Script

Execute the setup script and paste your Extension ID when prompted:

```bash
./setup.sh
```

The script will:
- Make `host.py` executable.
- Generate `com.local.gemini_agent.json` with absolute paths and your Extension ID.
- Create the Brave Native Messaging directory if it doesn't exist.
- Symlink the manifest into Brave's configuration folder.

### 3. Finalize

1. Refresh your Gemini tab (`https://gemini.google.com/`).
2. Open the browser console (F12) and wait for the log: `Agent ARMED`.

## Usage: The Autonomous Loop

To trigger the agent, simply include a JSON code block in your prompt or have Gemini generate one. The extension will automatically detect, execute, and reply.

### Example Command

```json
{
  "action": "run_shell",
  "command": "git status"
}
```

### Supported Actions

| Action | Description |
|--------|-------------|
| `run_shell` | Executes any bash command. Supports pipes and redirects. |
| `write_file` | Creates or overwrites a file at a specific path. |
| `read_file` | Reads the contents of a file at a specific path. |

#### `run_shell`

```json
{
  "action": "run_shell",
  "command": "ls -la ~/Development"
}
```

#### `write_file`

```json
{
  "action": "write_file",
  "filepath": "/tmp/hello.txt",
  "content": "Hello from Gemini!"
}
```

#### `read_file`

```json
{
  "action": "read_file",
  "filepath": "/tmp/hello.txt"
}
```

## CLI Testing (Without Browser)

You can test the agent actions directly from the terminal using `main.py`:

```bash
# Run a shell command
python3 main.py run_shell "echo 'Hello World'"

# Write a file
python3 main.py write_file /tmp/test.txt "File content here"

# Read a file
python3 main.py read_file /tmp/test.txt

# Interactive mode
python3 main.py interactive
```

## Debugging & Logs

If the "System Result" appears empty or the agent fails to respond:

- **Host Logs:** Check the real-time Python execution log:
  ```bash
  tail -f /tmp/gemini_host.log
  ```
- **Browser Console:** Open the DevTools (F12) on the Gemini tab to see payload detection and injection logs.
- **Extension Console:** Check the Background Page logs in `brave://extensions/` to verify Native Messaging connectivity.

### Common Issues

| Issue | Solution |
|-------|----------|
| `Specified native messaging host not found` | Verify the symlink in `~/.config/BraveSoftware/Brave-Browser/NativeMessagingHosts/` and ensure the Extension ID in `com.local.gemini_agent.json` is correct. |
| `Could not establish connection. Receiving end does not exist.` | Refresh the Gemini tab so the updated `content.js` is injected. |
| Service Worker shows "Inactive" | Click the "service worker" link in `brave://extensions/` to wake it up. |

## Security Considerations

- **Local Execution:** This agent has the same permissions as the user running the browser. Use with caution.
- **Allowed Origins:** The Native Messaging manifest restricts communication strictly to your specific Extension ID.
- **Command Injection:** Be mindful of the commands Gemini generates. The `run_shell` action executes commands directly on your system.
